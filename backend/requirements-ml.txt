# Heavy ML dependencies (Phase 2+). Kept out of requirements.txt so the web
# skeleton installs in seconds instead of pulling ~3 GB.

# TORCH MUST BE INSTALLED FIRST, from the CUDA-specific index — the default PyPI
# wheel is a different build and pip will not "upgrade" you onto the right one
# once a plain torch is resident. For this machine (RTX 3050, driver 610.62):
#
#     pip install torch==2.13.0 torchvision --index-url https://download.pytorch.org/whl/cu126
#     pip install -r requirements-ml.txt
#
# Pick your own index from https://pytorch.org/get-started/locally/ if the GPU
# or driver differs. Verify afterwards with:
#     python -c "import torch; print(torch.cuda.is_available(), torch.cuda.get_device_name(0))"

# VRAM BUDGET — this machine has 4 GB (RTX 3050 Laptop)

# 4 GB is the binding constraint on every model choice in this project, and it
# is why app/ml/registry.py enforces one resident model at a time.
#
#   Grounding DINO Swin-T   ~700 MB weights   ~2-3 GB inference   fits (tight)
#   SAM ViT-B               ~375 MB weights   ~2-3 GB inference   fits
#   SAM ViT-H               ~2.4 GB weights   ~6-8 GB inference   DOES NOT FIT
#   MobileSAM                ~40 MB weights     <1 GB inference   fits easily
#
# So: ViT-B, not the ViT-H that every tutorial defaults to. And Grounded SAM
# cannot hold DINO and SAM resident simultaneously (~5-6 GB) — it must run them
# sequentially, unloading between stages.
